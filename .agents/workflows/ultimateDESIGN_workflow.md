# UltimateDESIGN 循证式微更新平台工作流 (Agent Workflow)

> **定位**：此工作流定义了 `ultimateDESIGN` 系统的四大实验室数据串联机制，以及大语言模型代理、算力引擎（SD/ControlNet）之间的交互规则。

## 阶段一：多模态资产测度 (Page 1)
1. **地块向量化**：获取 `Key_Plots_District.json` 等矢量红线。
2. **MPI 打分执行**：调用 AHP 模型提取并校验专家权重矩阵，计算“空间-社会-环境”综合评估分值（MPI）。
3. **语义切图与存储**：对外部输入的非结构化文献（如《开题报告》、《保护名录》等）通过 RAG 模型切片并存入语料库，形成约束策略条件。

## 阶段二：全息空间诊断 (Page 2)
1. **多模态挂载**：挂载基于 CV 的空间视域测量结果（绿视率 GVI / 开阔度 SVF）和 基于 NLP 的公众社会情感（UGC）。
2. **体检聚合**：融合不同维度的弱项，通过雷达图与多层级指标在 3D 沙盘中叠加映射。
3. **输出诊断结论**：基于雷达最低项，抛出针对该街区/地块的结构性优化指令锚点（如“绿化渗透迫切”等），传递给下一流程。

## 阶段三：空间形态衍生模拟 (Page 3)
1. **生网底版获取**：获取带有街区肌理特征的底版（2D CAD路网/白模/现状实拍图）。
2. **参数翻译**：依据上一步诊断短板，设定 AIGC 模型的“规划算子权重”（如增加自然生态景观权重）。
3. **ControlNet 制图约束**：使用 Canny/MLSD 等预处理器锁死历史建筑结构的透视骨架和街区路网，生成带有强制“空间保护约束”的总平面图、立面意向或鸟瞰模拟。
4. **决策对撞比选**：对比衍生出的多种草图形态。

## 阶段四：LLM 多代理演进博弈 (Page 4)
1. **红线加载**：以“名城保护区严格保护法则”和“历史建筑强制保留”作为提示词的 System Prompt 安全墙（硬底座约束）。
2. **基于 RAG 的合规性校验**：检索阶段一线传入的《长春市总体规划》或《周边业态报表》切片。
3. **多主体辩论与输出方向**：促发“规划师”、“开发商”、“公众代表”围绕同一地块开展基于思维链（CoT）的逻辑博弈。
4. **平行演化导则输出**：LLM 强制融合讨论，输出 2-3 种截然不同、具备可实操性的专项发展策划建议或更新导则方案。
