## 4. 第一阶段详细执行方案（手段-方法-步骤-成果）

## 4.1 子任务 A：数据资产台账

### 手段

- 文件级清单扫描（CSV/JSON/GEOJSON/XLSX）
- 字段抽样与元数据登记
- 配置文件对照（`config.yaml`）

### 方法

- “资产卡片法”：每个数据集一张卡，记录来源、时间、字段、用途、限制
- “配置反查法”：从配置项反推必需数据，识别缺口

### 步骤

1. 枚举 `data/` 下所有数据文件
2. 抽取前 20 行识别字段结构
3. 建立台账字段：
   - 文件名、类型、行数、关键字段、更新时间、用途模块、可信等级
4. 标注每份数据对应页面/引擎：
   - 页面 1（MPI/管理）
   - 页面 2（诊断可视化）
   - 页面 4（RAG/协商）

### 成果

- 《数据资产台账》（建议 `xlsx` + `md` 双版本）
- 一页式“数据流向图”（Data -> Engine -> Page）

### 验收标准

- 台账覆盖率 100%
- 每份数据至少标注 1 个消费模块
- 缺失关键数据明确标红

---

## 4.2 子任务 B：数据质量报告

### 手段

- Python/Pandas 统计
- 字段约束检查
- 空间字段边界检查（经纬度范围）

### 方法

- “四类体检法”：
  1) 完整性：空值率、缺字段
  2) 一致性：命名、编码、类型
  3) 合理性：取值范围与业务逻辑
  4) 可用性：是否满足当前模型输入

### 步骤

1. 对每个 CSV 计算：
   - 行数、列数、空值占比、重复率
2. 关键字段校验：
   - POI/交通：`Lat`,`Lng` 范围是否在长春周边
   - NLP：`Text`,`Source` 是否可用
   - GVI：`ID`,`GVI`,`SVF`,`Enclosure`,`Clutter` 是否齐全
3. 异常样本登记：
   - 坐标缺失、文本为空、编码异常
4. 输出修复建议：
   - 补字段、重命名、去重、重新采样

### 成果

- 《数据质量报告》
- 《问题清单与修复优先级》

### 验收标准

- 每份数据都有质量评分（A/B/C）
- 每类问题都有可执行修复动作
- 明确“阻塞中期答辩”的高优先问题

---

## 4.3 子任务 C：诊断证据包索引

### 手段

- 指标图件模板整理
- 页面输出映射（页面->图件）
- 视频镜头需求反推

### 方法

- “证据链映射法”：每张图都能回答一个问题
- “图纸-视频同源法”：图纸素材直接进入视频镜头

### 步骤

1. 定义证据包目录：
   - 现状问题证据
   - 模型诊断证据
   - 策略推演证据
2. 为每类证据建立索引项：
   - 文件名、数据来源、生成脚本、应用场景
3. 建立“任务书图纸要求”映射：
   - 遗产热力图 -> 数据输入 -> 生成流程
   - 交通图/结构图 -> 数据输入 -> 生成流程

### 成果

- 《诊断证据包索引表》
- 视频镜头素材清单（按时长分段）

### 验收标准

- 至少覆盖任务书中全部“必需图纸类型”
- 每项证据可追溯到数据文件与生成方法
