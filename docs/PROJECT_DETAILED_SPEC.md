> **文档密级**：学术内部公开

## 1. 系统架构综述与理论支撑

本系统依托“物理-社会”二元空间诊断理论，构建了一个具备自适应能力的决策闭环。技术链路横跨地理信息处理、计算机视觉测度、生成式设计模拟与提示词工程决策。

### 1.1 全链条业务逻辑（Evidence-to-Action）

1. **多源底版汇聚 (Evidence Acquisition)**：  
   整合 WGS84 投影坐标系下的路网矢量、POI 点位快照及百度街景 (Baidu SV) 影像流。
2. **多维指标测度 (Multi-metric Measurement)**：  
   计算空间潜力分级、人本街道指数（GVI/SVF 等）及公众情感极值。
3. **推演预演闭环 (Scenario Simulation)**：  
   在法定规划导则约束下，利用潜在特征控制网络进行针对性立面修缮模拟。
4. **决策协调共识 (Collaborative Decision)**：  
   基于大语言模型 (LLM) 进行三方利益主体协商，生成具象化的空间织补建议。

## 2. 空间评价模型与数学描述

### 2.1 多维更新潜力指数测度 (MPI)

**模型功能**：通过评价区域现状物质老化程度与社会需求密度的重合度，确定微更新的实施优先级。

#### 核心公式：
\[
MPI_i = \frac{w_{space}\cdot S_i + w_{social}\cdot D_i + w_{env}\cdot(1-E_i)}{w_{space}+w_{social}+w_{env}+\epsilon}\times 100
\]

#### 变量语义定义：
- \(S_i\)：**物理空间潜力原分**。反映地块容积效率与功能混合度的冗余深度。
- \(D_i\)：**社会需求响应强度**。基于热力人口分布与公共服务设施覆盖度的综合测定值。
- \(E_i\)：**环境现状品质评分**。基于规划局历史街区现场勘测台账的标准化分值。
- \(1-E_i\)：**环境干预紧迫度**。用于量化环境病灶对更新行为的反向拉动力。
- \(w\)：**专家权重矩阵**。通过 AHP 层次分析法在交互侧实时标定。

---

### 2.2 基于 AHP 的判断矩阵归一化逻辑

为确保专家经验能够科学转化为权重分布，系统执行以下归一化过程：
\[
\mathbf{w}' = \frac{[w_{space}, w_{social}, w_{env}]}{\sum w + \epsilon}
\]
生成相对重要性矩阵 \(A_{ij}\)，并确保在特征向量计算后其一致性比例 \(CR < 0.1\)，以通过数学校验逻辑。

---

### 2.3 街道空间形态 CV 测度指数

基于 SegFormer 语义分割框架，对 150 公顷研究范围内的街景采样点进行全要素解析。

- **绿视率 (GVI)**：\( GVI = \frac{\sum P_{vegetation}}{\sum P} \times 100 \)，反映街道生态遮蔽力。
- **天空开敞度 (SVF)**：\( SVF = \frac{\sum P_{sky}}{\sum P} \times 100 \)，表征街道微气候通风与视觉压抑感。
- **空间围合度 (Enclosure)**：\( \frac{\sum (P_{building}+P_{wall}+P_{veg})}{\sum P} \)，反映空间立面的包裹效果。
- **视觉杂乱度 (Clutter)**：\( \frac{\sum (P_{pole}+P_{sign}+P_{fence})}{\sum P} \)，衡量街道设施配置的视觉冗余。

## 3. 动态图层可视化映射逻辑

### 3.1 基于 BI 梯度色彩的数值感知
系统采用自适应线性插值器（Linear Interpolator），根据测度项 \(v\) 的极值区划（\(v_{min}, v_{max}\)），将归一化分值映射至 **[Cyan - Purple - Pink]** 三段式现代色彩梯度，以提升非专业决策者对空间病灶的视觉辨识敏感度。

---

## 4. 模块化解析引擎规格

### 4.1 方案模拟引擎 (Stable Diffusion + ControlNet)
- **约束机制**：利用 Canny/Depth 预处理器，提取现状历史建筑立面的“先验格局”，在不破坏历史肌理的基础上进行纹理织补。
- **参数控制**：通过“规划算子”调节景观、历史、现代三个维度的语义权重，确保渲染结果符合《历史文化名城保护条例》的弹性控制要求。

### 4.2 决策博弈引擎 (Multi-Agent RAG)
- **核心逻辑**：引入检索增强生成 (RAG) 技术，将长春当地的规划法规（如《长春市城市更新管理办法》）向量化后接入博弈流程。
- **主体模拟**：构建“公众代表、市场投资者、专业规划师”三方 Prompt 模板，执行基于思维链 (CoT) 的逻辑推理，以达成利益最大公约数。

## 5. 系统异常冗余与鲁棒性设计

1. **算力降维机制**：在本地 SD/Ollama 引擎处于未载入状态时，自动激活“演示序列”，利用预存的仿真数据流维持交互闭环。
2. **数据编码鲁棒**：支持 utf-8-sig 与 gbk 双重动态适配，确保不同批次的多源数据表在全息诊断模块中不发生解析溢出。
