## 项目总结报告（`ultimateDESIGN`）

这个项目是一个**以 Streamlit 为核心的城市规划/设计决策平台**，聚合了数据分析、数字孪生可视化、AIGC 生成和 LLM 协同决策能力。整体形态是**单仓库、多页面应用 + 离线数据处理脚本**。

### 一、项目结构与职责

- `app.py`：应用主入口与页面总导航  
- `pages/`：核心业务页面  
  - `1_数据底座与规划策略.py`：数据治理、策略评估  
  - `2_数字孪生与全息诊断.py`：2D/3D 场景与指标诊断  
  - `3_AIGC设计推演.py`：AIGC/ControlNet 设计推演  
  - `4_LLM博弈决策.py`：多主体博弈与政策建议  
- `src/engines/`：业务引擎层（NLP、CV、爬取、LLM/SD 接口）  
- `src/ui/`、`src/utils/`：UI 组件与工具函数  
- `data/`：业务数据与中间结果（POI、交通、NLP、GVI、RAG 等）  
- `tools/`：采集、推理、环境检查等离线脚本  
- `assets/`：静态资源与地图相关文件  

### 二、技术栈识别

- **语言**：Python  
- **应用框架**：Streamlit  
- **可视化/空间**：PyDeck、Plotly、Folium  
- **AI 能力**：Torch、Transformers；外接 Ollama、Stable Diffusion WebUI  
- **依赖管理**：`requirements.txt`（pip），运行脚本偏 Conda 环境（如 `run.bat`、`run_portable.bat`）

### 三、配置与运行方式

- 核心配置：`config.yaml`（模型地址、路径等）  
- Streamlit 配置：`.streamlit/config.toml`  
- 环境变量模板：`.env.example`  
- 启动方式：`streamlit run app.py`（文档与批处理脚本均指向该方式）  
- 便捷启动：`run.bat` / `run_portable.bat`（含环境探测与端口处理逻辑）

### 五、主要风险
