# UltimateDESIGN

**长春伪满皇宫周边街区微更新与城市设计决策支持平台**

UltimateDESIGN 是一个面向城乡规划专业城市设计课程、毕业设计和研究展示的 Streamlit 应用。项目严格按 **13 个专业阶段** 组织页面，贯穿了「前期现状分析 -> 中期概念策略 -> 后期设计深化与导出」的完整循证规划工作流。

---

## 🌟 核心特色与架构基座

- **13 阶段独立工作流**
  将城市设计拆解为 13 个标准化步骤，每个步骤均封装了独立的功能面板、AI 图纸提示词生成逻辑与阶段答辩总结。
- **41 种专业图纸模板覆盖**
  内置 41 种基于 Stable Diffusion 和 ControlNet 的提示词模板，系统会自动提取空间资产（如建筑高度、POI 活力）注入并格式化提示词。
- **动态驱动的答辩图表**
  利用 Plotly 为 13 个阶段的答辩小结自动生成强相关的分析图表，包含 MPI 更新潜力排行、多主体共识度雷达、分期实施甘特图等。
- **LLM 循证推演逻辑**
  深度挂载本地 Gemma 模型，通过跨阶段数据总线 (`stage_data_bus`)，一键生成去 AI 味、专业术语饱满的学术推理段落。
- **极速渲染与局部碎片刷新**
  采用基于时效的数据格式缓存优化（`@st.cache_data`），配合 JS 异步地图挂载与 Streamlit 碎片化重绘（`@st.fragment`），保障 150 公顷尺度基底的丝滑交互。

## 🚀 快速启动 (Quick Start)

这份说明旨在为您提供**最短路径**将本项目在本地跑通。

### 📦 1. 环境准备与依赖安装

请确保您的计算机已安装 Python 3.10+。打开终端，克隆代码库并安装核心依赖：

```powershell
# 1. 抓取代码
git clone https://github.com/Chain813/ultimate-design.git
cd ultimate-design

# 2. 安装依赖
pip install -r requirements.txt
```

### ▶️ 2. 启动总控制台

无需预先配置任何复杂的 API Key，您可以直接通过 Streamlit 唤醒系统界面：

```powershell
streamlit run app.py
```

服务启动后，您的默认浏览器将自动打开并访问：
`http://localhost:8501/`

### 🧭 3. 基础页面导航机制

系统取消了传统的侧边栏导航，所有的模块跳转均在 **顶部导航栏 (Top Nav)** 完成。
请按照 `[01]` 到 `[13]` 的编号顺序，依次体验城市设计的完整推演流。

### ✅ 4. 健康自检 (Smoke Test)

如果您修改了任何代码，请在重新启动前运行以下命令以确保系统状态健康：

```powershell
# 语法合规性检查
python -m compileall app.py pages src tests tools

# 自动化测试引擎校验
pytest

# 启动核心环境监测
python tools/startup_smoke.py
```

## 🔧 安装与算力挂载指南 (Installation Guide)

要激活 UltimateDESIGN 的「完全体」形态（包括 AIGC 推演与 LLM 博弈），您需要将本地的算力基础设施与应用进行挂载。

### 🧠 1. 配置云端大模型 API (DeepSeek)

平台已全面接入 DeepSeek 官方云端 API，为您提供最顶级的推理能力，**且完全不占用本地计算资源（零显存消耗）**。

1. 前往 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册并获取您的 API Key。
2. 在项目根目录找到或创建 `.env` 文件。
3. 填入您的 API 密钥：
   ```env
   DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxxxxx"
   ```
4. **验证机制**：系统启动后，底层的 `llm_engine` 会自动读取该环境变量并在执行案例推演、多方博弈时将请求发送至云端处理。

### 🎨 2. 挂载视觉渲染引擎 (Stable Diffusion WebUI)

平台内置的 41 种图纸模板需要依托 SD 的强大生图能力及 ControlNet 约束机制。

1. 启动您的本地 SD WebUI 环境（如秋叶整合包或纯净版 WebUI）。
2. **核心步骤**：必须开启 API 监听模式。在您的启动脚本（如 `webui-user.bat`）中添加以下参数：
   ```bat
   set COMMANDLINE_ARGS=--api --listen
   ```
3. **验证机制**：确保 SD 运行在默认端口 `127.0.0.1:7860`。回到平台主页，此时视觉渲染引擎 HUD 指示灯将显示「已联机」。

### 📂 3. 空间数据与资产管理

如果您希望将平台迁移到全新的设计地块，只需替换 `data/` 目录下的底层数据即可，**代码层无需任何修改**。

```text
data/
├── shp/
│   ├── Boundary_Scope.geojson       --- 研究红线底图
│   ├── buildings.geojson            --- 建筑基底（必须包含 Floor 高度字段）
│   └── Key_Plots_District.json      --- 5 个重点更新地块边界
├── streetview/
│   └── (您的调研相片库)              --- 街景调研数据，建议按命名规范存储
└── ...
```

## 👶 城市规划师新手教程 (Beginner Guide)

初次使用 UltimateDESIGN 时，面对众多的面板和专业图表可能会略感困惑。
请跟随本向导，我们将手把手带您体验一次完整的「循证驱动型街区微更新设计推演」。

### 🟢 阶段一：前期诊断与评估入局

作为规划师，第一步是建立对场地的客观认知框架。

1. **解读任务书 (Page 01)**：通过顶部的数字导航进入 `01_任务解读`。系统会帮您梳理任务书里的红线限制，并提供初始区位图提示词。
2. **沉浸式街景与 3D 底座 (Page 03 & 04)**：前往 `04_现状分析`。您将看到巨大的 WebGL 渲染地球，搭载了该片区真实的 150 公顷建筑底座。打开左侧的「光照推演」，观察不同时刻建筑阴影。
3. **识别潜力黑马 (Page 05)**：翻转至 `05_问题诊断`，查看「MPI 更新潜力排行榜」与「地块诊断雷达图」。这套 AHP-MPI 算法会帮您揪出亟需改造的地段。

### 🟡 阶段二：价值斡旋与策略拟定

这是整个系统最具 AI 智慧的模块：如何平衡各方利益？

1. **寻找参照系 (Page 06)**：在 `06_目标定位` 面板，召唤大语言模型进行「案例对标」，系统会自动比对成功更新案例的底层逻辑。
2. **三主体桌游博弈 (Page 07)**：规划不是“一言堂”。进入 `07_设计策略`。开启「多主体推演」引擎。您将目睹 AI 扮演的多方角色进行激烈文本争论。博弈结束后，会沉淀出一张「多主体共识度雷达图」。

### 🔴 阶段三：视觉转化与归档出库

理清逻辑后，是时候输出震撼人心的专业图纸了。

1. **概念生形 (Page 08)**：在 `08_总体城市设计` 中调取总平图生形面板。系统组装出精准的 AIGC 指令发送至 Stable Diffusion，生成概念规划总平图。
2. **前后对比魔术 (Page 10)**：选取重点地段，进入 `10_重点地段深化`。放置现场破败街景图，推演未来建成效果图 (After)。
3. **收网导出 (Page 13)**：前往 `13_成果表达`。这里罗列了 41 款图纸提示词模板。点击生成学术小结，平台会将前 12 个阶段数据流熔炼为一篇结构严密的学术终评报告。

## 🗂️ 核心文件结构与代码规范 (Project Structure)

### 核心目录架构

```text
ultimateDESIGN/
├─ 🏠 app.py                       # 首页、平台状态、全局地图 2D/3D 基底入口
├─ 🧭 pages/                       # 核心视窗：文件名决定侧栏呈现与路由执行顺序
├─ 🧬 src/                         # 领域代码：核心逻辑与可复用套件
│  ├─ ⚙️ config/                   # 静态变量、路径注册、配置加载
│  ├─ 🧠 engines/                  # 计算、大模型、AIGC 逻辑层（严禁在此写 UI）
│  ├─ 🎛️ ui/                       # 全局外壳、原子设计系统、图表配色方案
│  ├─ 🧰 utils/                    # IO 操作、数据转换、文本清洗等工具函数
│  └─ 🧩 workflow/                 # 13 阶段工作流映射引擎与数据总线
├─ 🎨 assets/                      # 前端静态物料：CSS 样式、HTML WebGL 模板
├─ 🗃️ data/                        # 数据仓库：空间底座、表格、语义分析暂存文件
├─ 📄 docs/                        # 规划文书：任务书、开题报告、法定政策资料
├─ 🌐 static/                      # Streamlit Server 暴露的网络静态资源
├─ 🛠️ tools/                       # DevOps：环境检查与 Smoke 启动测试脚本
└─ ✅ tests/                       # 自动化验证：Pytest 单元测试
```

### 🧭 页面路由与 13 阶段映射 (pages/)

- **前期**：`01_任务解读.py` 到 `05_问题诊断.py`（数据获取与现状分析）
- **中期**：`06_目标定位.py` 到 `07_设计策略.py`（概念生成与应对策略）
- **后期**：`08_总体城市设计.py` 到 `13_成果表达.py`（设计生成与成果展示导出）

### ⚡ 性能优化与渲染架构

- **统一数据流式静默缓存 (`@st.cache_data`)**: 对空间数据 IO 和高频耗时运算做缓存。绝对禁止使用 `show_spinner=False` 防止打破向下兼容。
- **地图组件静态渲染 (`@st.fragment`)**: 确保主地图的丝滑交互，切换图层或光照时仅重绘该地图块，绝不导致整个 App 重启。
- **异步大模型推演**: 将复杂 AI 生成推送到后台以确保界面响应度。

### 🛠️ 底线维护规则

1. 路由跳转由 `app_shell` 中的导航栏统一接管。
2. 坚决避免在 `pages/` 内书写复杂的业务计算与模型调用，一律写到 `src/engines/`。
3. UI 优化提炼至 `src/ui/design_system.py`，禁止在页面复制冗长的 CSS Markdown。
4. 任何核心逻辑的变动，提交前至少执行语法检查和 Pytest 测试。
