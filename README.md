# UltimateDESIGN

**长春伪满皇宫周边街区微更新与城市设计决策支持平台**

UltimateDESIGN 是一个面向城乡规划专业城市设计课程、毕业设计和研究展示的 Streamlit 应用。项目严格按 **13 个专业阶段** 组织页面，贯穿了「前期现状分析 -> 中期概念策略 -> 后期设计深化与导出」的完整循证规划工作流。

---

### 🌟 核心特色与架构基座

- **13 阶段独立工作流**
  将城市设计拆解为 13 个标准化步骤，每个步骤均封装了独立的功能面板、AI 图纸提示词生成逻辑与阶段答辩总结。
- **41 种专业图纸模板覆盖**
  内置 41 种基于 Stable Diffusion 和 ControlNet 的提示词模板，系统会自动提取空间资产（如建筑高度、POI 活力）注入并格式化提示词。
- **动态驱动的答辩图表**
  利用 Plotly 为 13 个阶段的答辩小结自动生成强相关的分析图表，包含 MPI 更新潜力排行、多主体共识度雷达、分期实施甘特图等。
- **LLM 循证推演逻辑**
  深度挂载本地 Gemma 模型，通过跨阶段数据总线 (`stage_data_bus`)，一键生成去 AI 味、专业术语饱满的学术推理段落。
- **极速渲染与局部碎片刷新**
  采用基于时效的数据格式缓存优化（`@st.cache_data`），配合 JS 异步地图挂载与 Streamlit 碎片化重绘（`@st.fragment`），保障 150 公顷尺度基底的丝滑交互。

### 🧭 核心业务流：三大板块与 13 阶段页面

#### 🔹 前期：数据获取与现状分析 (01 - 05)
- **01. 任务解读**：剖析项目概况与开题报告，生成初始区位图提示词，自动统计任务书工作量分配占比 (Pie)。
- **02. 资料收集**：驱动语义萃取引擎 (MarkItDown) 与空间数据资产管理，量化数据资产完备度评分 (Bar)。
- **03. 现场调研**：沉淀街景样本库并支持四向照片检索，实时追踪调研任务完成度 (Stack Bar)。
- **04. 现状分析**：渲染 3D 空间底座，聚合 POI 活力、建筑形态与天际线特征，输出建筑高度分布 (Pie)。
- **05. 问题诊断**：基于 AHP-MPI 评估模型生成地块诊断雷达与 AI 报告，输出 MPI 更新潜力排行榜 (Bar)。

#### 🔹 中期：概念生成与应对策略 (06 - 07)
- **06. 目标定位**：通过 LLM 进行案例对标与总体愿景提炼，梳理目标体系层级分布 (Treemap)。
- **07. 设计策略**：模拟居民、开发商与规划师的三方博弈协商，结合 RAG 政策校验绘制多主体共识度雷达 (Polar)。

#### 🔹 后期：设计生成与成果表达 (08 - 13)
- **08. 总体城市设计**：AIGC 概念总平面图生形及空间推演，输出规划用地结构占比 (Pie)。
- **09. 专项系统设计**：叠合交通、公共空间、风貌与历史文化四大专项，量化专项系统设计覆盖度 (Bar)。
- **10. 重点地段深化**：聚焦 5 类重点地块，执行 AIGC 街景前后推演 (Before/After)，对比重点地块综合指标 (Polar)。
- **11. 实施路径**：定义六类更新方式并排布近中远三期实施时间轴，生成分期实施面积占比 (Gantt Bar)。
- **12. 城市设计导则**：LLM 导则生成汇总与空间管控指标设定，考核导则管控指标覆盖度 (Bar)。
- **13. 成果表达**：全流程 41 种图纸模板总览与多格式图册统一导出，统计图纸提示词模板阶段分布 (Bar)。

### 🧩 核心底层引擎 (src/)

- **`workflow/stage_data_bus.py` (跨阶段数据总线)**
  管理全链条证据传递，保障后一阶段可无缝调用前一阶段的统计与诊断结论。
- **`ui/module_summary.py` (阶段答辩小结组件)**
  为全部 13 个页面底部注入学术总结面板、动态 Plotly 图表与 Gemma 一键推演按钮。
- **`ui/app_shell.py` (全局 UI 外壳)**
  负责全宽沉浸式 Top Nav 导航映射与状态指示器呈现。
- **`engines/drawing_prompt_templates.py` (提示词模板中心)**
  注册并严格管理 41 种专业图纸的系统指令与生成规范。
- **`engines/spatial_engine.py` (空间计算核心)**
  极速执行 GeoJSON 处理、天际线计算和 GVI 指标挂载，并已全面实现高可用静默缓存。

### 🚀 启动与验证规范

**1. 部署依赖**
```bash
pip install -r requirements.txt
```

**2. 启动总台**
```bash
streamlit run app.py
```
*(注：需本地启动 Ollama 引擎并挂载 `gemma4:e2b-it-q4_K_M` 以激活完全体 AI 体验)*

**3. CI/CD 测试与健康检查**
```bash
# 执行代码合规性编译检查
python -m compileall app.py pages src tests tools

# 运行自动化单元测试
pytest

# 运行启动文件健康自检 (Smoke Test)
python tools/startup_smoke.py
```
