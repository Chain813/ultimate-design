# 🗺️ 城乡规划专业 — 技术术语通俗解释

对于习惯了 ArcGIS、AutoCAD 控规绘图和城市设计建模的规建学子来说，编程世界的术语可能听起来很陌生。这份指南用**城乡规划**专业的视角，帮你快速理解这些“黑话”。

---

### 1. Python (编程语言)
- **规划类比**：**城市底层的运行逻辑或 GIS 的高级计算引擎**。
- **解释**：它就像是规划编制背后的数学模型。如果你把 CAD 看作画图工具，把 GIS 看作点线面展示工具，那么 Python 就是能让你通过写指令来处理“全市域 POI 数据”、进行“路网通达性分析”或自动生成“更新潜力评估”的超级大脑。

### 2. 环境 / Virtual Environment (虚拟环境)
- **规划类比**：**针对特定规划项目的“纯净工作空间”**。
- **解释**：做“总体规划”可能需要一套分析工具，做“详细规划”或“专项研究”又需要另一套插件。
- **“环境”** 就是给每个规划项目单独开辟的一个“数字办公室”，里面只装这个项目需要的库（如处理地理数据的库），防止不同项目间的工具版本冲突，保证分析结果的准确性。

### 3. 依赖 / Dependencies (库)
- **规划类比**：**ArcGIS 的工具箱 (Toolbox) 或 Grasshopper 的分析插件**。
- **解释**：Python 本身功能有限，但它可以调用别人写好的“规划工具包”。比如：
    - `pandas`：像是一个能瞬间处理几十万行人口或土地数据的超级 Excel。
    - `geopandas`：像是一个不需要打开 GUI 界面就能进行空间叠合分析的“代码版 GIS”。
    - `pydeck`：像是一个专门负责在网页上渲染 3D 城市动态大数据（如人流热力图）的引擎。

### 4. Requirements.txt (依赖清单)
- **规划类比**：**规划任务书中的“技术标准清单”或“软件环境要求”**。
- **解释**：记录了运行这个项目所需的全部“分析包”及其版本。有了它，其他规划师在自己电脑上输入一行命令，就能瞬间配齐和你一模一样的分析环境。

### 5. Streamlit (交互框架)
- **规划类比**：**一个动态交互的“数字规划展示看板”**。
- **解释**：通常代码只输出静态图表。Streamlit 能把你的分析过程变成一个网页。在这个页面上，你可以拉动滑动条改变“容积率”或“绿地率”，地图上的诊断结果和潜力评分会实时跳动，非常适合向评审会或公众进行方案推演展示。

### 6. Git / GitHub (版本控制与仓库)
- **规划类比**：**规划成果的版本追溯 (从“初稿”到“报批稿_2024_04_26”)**。
- **解释**：
    - **Git**：像是一个“时光机”，记录代码的每一次改动。
    - **GitHub**：像是一个云端的“规划局资料库”，大家把代码放在这里备份、交流和协作开发。

### 7. RAG (检索增强生成)
- **规划类比**：**一位熟背《城市规划强制性条文》和《各省市更新导则》的智能助理**。
- **解释**：一般的 AI 可能会乱编政策。RAG 技术是让 AI 先去翻阅你准备好的 PDF（如本项目 docs 文件夹里的规范），然后再基于这些真实依据来回答问题，确保它不脱离规划现实。

### 8. API (接口)
- **规划类比**：**获取百度地图或高德地图数据的“数字通行证”**。
- **解释**：如果你想获取长春市的实时交通流量。不需要自己去路口数车，只需通过地图公司提供的“插口”（API）发个请求，数据就会自动流进你的分析模型。
