# 🏛️ 城乡规划与建筑学专业 — 技术术语通俗解释

对于习惯了 CAD、SketchUp 和 Rhino 的规建学子来说，编程世界的术语可能听起来很陌生。这份指南用我们专业的视角，帮你快速理解这些“黑话”。

---

### 1. Python (编程语言)
- **建筑类比**：**Rhino 的核心引擎**。
- **解释**：它就像是软件背后的逻辑引擎。如果你把 CAD 看作一个只能画图的工具，那么 Python 就是一个能让你通过写指令来生成建筑、分析交通或处理上万条数据的“超级插件”。

### 2. 环境 / Virtual Environment (虚拟环境)
- **建筑类比**：**针对不同项目的“插件配置方案”**。
- **解释**：在做甲项目时，你可能需要 Rhino 6 + Grasshopper A 插件；做乙项目时，可能需要 Rhino 7 + Grasshopper B 插件。如果你把它们装在一起，可能会冲突报错。
- **“环境”** 就是给每个项目单独开辟的一个“纯净工作室”，里面只装这个项目需要的工具，互不干扰。

### 3. 依赖 / Dependencies (库)
- **建筑类比**：**Grasshopper 的运算器插件包 (如 Kangaroo, Ladybug)**。
- **解释**：Python 本身功能有限，但它可以调用别人写好的工具包（库）。比如：
    - `pandas`：像是一个能处理无限行数据的超级 Excel。
    - `plotly`：像是自动帮你画雷达图、分析图的“绘图员”。
    - `pydeck`：像是专门负责渲染 3D 城市地图的“渲染引擎”。

### 4. Requirements.txt (依赖清单)
- **建筑类比**：**施工图中的“材料表”或“插件安装清单”**。
- **解释**：这是一个文本文件，记录了这个项目运行所需要的全部“插件”及其版本。通过这个清单，电脑可以一键配齐所有工具，不需要你一个个去搜索安装。

### 5. Streamlit (交互框架)
- **建筑类比**：**一个可以自动互动的“电子汇报展板”**。
- **解释**：通常写代码只能在黑窗口看结果。Streamlit 能把你的分析代码瞬间变成一个网页。在这个网页上，你可以拉动滑动条改变参数，地图和图表会跟着实时变动，非常适合向甲方或导师汇报展示。

### 6. Git / GitHub (版本控制与仓库)
- **建筑类比**：**图纸的版本管理 (从“方案一”到“最终版_绝对不改_v3”)**。
- **解释**：
    - **Git**：像是一个“存档机器”，它能记录你代码的每一次修改，随时可以“读档”重来。
    - **GitHub**：像是一个云端的“项目部”，大家把代码放在这里共享、备份和协作。

### 7. RAG (检索增强生成)
- **建筑类比**：**翻着《规范》和《任务书》帮你写方案的助理**。
- **解释**：一般的 AI (如 ChatGPT) 可能会乱编。RAG 技术是让 AI 先去读你提供的 PDF（比如《长春市城市更新条例》），然后再根据读到的准确内容回答问题，确保它不瞎说。

### 8. API (接口)
- **建筑类比**：**软件的“数据插口”**。
- **解释**：比如你想在 Rhino 里调取百度地图的实时路况。你不需要自己去测绘，只需通过百度提供的“插口”（API）发个请求，它就会把数据传给你。
